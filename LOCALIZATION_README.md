# Система локализации

## Установка зависимостей

Перед использованием системы локализации необходимо установить зависимости:

```bash
npm install
```

Это установит следующие пакеты:
- `i18next` - основная библиотека для интернационализации
- `react-i18next` - интеграция i18next с React
- `i18next-browser-languagedetector` - автоматическое определение языка браузера

## Как это работает

### 1. Статический контент (UI элементы)

Статический контент интерфейса (кнопки, меню, заголовки) переводится через файл `src/i18n/config.js`. 
Все переводы уже добавлены для 6 языков:
- Русский (ru)
- Английский (en)
- Немецкий (de)
- Испанский (es)
- Французский (fr)
- Шведский (sv)

### 2. Динамический контент (объявления клиентов)

Пользовательский контент (названия недвижимости, описания, локации) автоматически переводится через API MyMemory Translation.

**Как это работает:**
- При смене языка система автоматически переводит:
  - Название недвижимости (`name`)
  - Локацию (`location`)
  - Описание (`description`)

- Переводы кэшируются для оптимизации производительности
- Если API недоступен, отображается исходный текст

## Использование в коде

### Перевод статического контента

```jsx
import { useTranslation } from 'react-i18next'

function MyComponent() {
  const { t } = useTranslation()
  
  return <h1>{t('home')}</h1> // "Главная" на русском, "Home" на английском
}
```

### Автоматический перевод пользовательского контента

```jsx
import { useAutoTranslate } from '../hooks/useAutoTranslate'

function PropertyCard({ property }) {
  const { translate, translateObj } = useAutoTranslate()
  
  // Перевести один текст
  const translatedName = await translate(property.name, 'ru')
  
  // Перевести объект
  const translatedProperty = await translateObj(
    property, 
    ['name', 'location', 'description'], 
    'ru'
  )
}
```

## API для автоматического перевода

Система использует **MyMemory Translation API** (бесплатный):
- До 10,000 символов в день бесплатно
- Не требует API ключа
- Работает через публичный endpoint

### Альтернативные варианты

Если нужен более мощный сервис, можно заменить в `src/services/translationService.js`:

1. **Google Cloud Translation API** (платный, но более точный)
2. **DeepL API** (платный, лучшее качество)
3. **Azure Translator** (платный)

## Добавление новых переводов

Чтобы добавить новый ключ перевода:

1. Откройте `src/i18n/config.js`
2. Добавьте ключ во все языки в объекте `resources`:

```javascript
resources: {
  ru: {
    translation: {
      myNewKey: 'Мой новый текст'
    }
  },
  en: {
    translation: {
      myNewKey: 'My new text'
    }
  },
  // ... остальные языки
}
```

3. Используйте в компоненте: `{t('myNewKey')}`

## Производительность

- Переводы статического контента загружаются мгновенно (из памяти)
- Переводы пользовательского контента кэшируются
- Перевод происходит асинхронно, не блокируя UI
- При первой загрузке может быть небольшая задержка при переводе большого количества текста

## Отладка

Для включения режима отладки i18next, измените в `src/i18n/config.js`:

```javascript
i18n.init({
  debug: true, // Включить отладку
  // ...
})
```

Это покажет в консоли информацию о загрузке переводов и отсутствующих ключах.
