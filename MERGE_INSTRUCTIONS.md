# Инструкция по разрешению конфликтов при merge

## Автоматическое разрешение конфликтов

Выполните следующие команды в терминале (Git Bash или CMD):

```bash
# 1. Переключитесь на ветку main
git checkout main

# 2. Обновите ветку main
git pull origin main

# 3. Выполните merge с автоматическим разрешением конфликтов в пользу ветки andrej
git merge andrej -X theirs --no-edit

# 4. Если все еще есть конфликты, принудительно используйте версию из andrej
git checkout --theirs .
git add .
git commit -m "Merge andrej into main - resolved conflicts"

# 5. Отправьте изменения в main
git push origin main

# 6. Вернитесь на ветку andrej
git checkout andrej
```

## Альтернативный способ (через GitHub)

Если автоматическое разрешение не работает, вы можете:

1. Откройте Pull Request на GitHub
2. Нажмите "Resolve conflicts" (если доступно)
3. Или используйте команды из раздела выше локально
4. После разрешения конфликтов, закоммитьте и запушьте изменения

## Важно

- Все конфликты будут разрешены в пользу версии из ветки `andrej`
- Файл `server/database.sqlite` - это бинарный файл базы данных, его лучше исключить из merge или использовать версию из `andrej`
