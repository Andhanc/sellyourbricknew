# Исправление ошибки "spawn ps ENOENT"

## Проблема:
```
Error: spawn ps ENOENT
```

`concurrently` пытается использовать команду `ps` (process status), которой нет в Docker-контейнере Railway.

## Решение:
Добавлен `procps` в `nixpacks.toml` - это пакет, который содержит утилиту `ps`.

## Что было исправлено:
- ✅ Добавлен `procps` в секцию `[phases.setup]` в `nixpacks.toml`

Теперь `concurrently` сможет использовать `ps` для отслеживания процессов.
